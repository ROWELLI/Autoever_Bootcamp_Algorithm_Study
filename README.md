# 알고리즘 스터디

누구나 각자 문제를 풀고 끝내는 방식이 아니라,  
정해진 주차에 맞춰 문제를 풀이하고 PR을 통해 함께 관리하는 알고리즘 스터디입니다.

스터디원은 **본인 이름의 브랜치에서 작업**한 뒤,  
마감 전까지 **팀장에게 Pull Request(PR)** 를 보내면 됩니다.  
PR 확인 후 팀장이 `main` 브랜치에 merge 합니다.

---

## 진행 방식

- 공용 레포지토리에서 스터디를 진행합니다.
- 각 스터디원은 **본인 이름 브랜치**를 생성하여 사용합니다.
- 매주 정해진 문제를 풀고, 본인 브랜치에 업로드합니다.
- 이후 **PR을 생성**하여 팀장에게 merge 요청을 보냅니다.
- 팀장은 제출된 PR을 확인한 뒤 `main` 브랜치에 merge 합니다.

---

## 브랜치 규칙

### 브랜치명
- 브랜치명은 **본인 이름**으로 생성합니다.

예시:
- `soyun`
- `minji`
- `jisu`

> 브랜치는 스터디 시작 후 한 번 만들고 계속 사용합니다.

---

## 제출 마감

- **매주 수요일 오후 6시까지** PR을 보낸 경우만 해당 주차 제출로 인정합니다.
- 마감 시간을 넘긴 PR은 **다음 주 제출** 또는 **지각 제출**로 처리할 수 있습니다.
- 세부 처리 방식은 스터디 운영 상황에 따라 조정할 수 있습니다.

---

## 폴더 구조

문제 풀이는 아래 구조를 기준으로 업로드합니다.

```bash
week01/
├─ 01_문제이름/
│  ├─ soyun.cpp
│  └─ minji.py
├─ 02_문제이름/
│  ├─ soyun.cpp
│  └─ minji.cpp
```

## 업로드 규칙

- 주차별 폴더를 먼저 생성합니다.
- 각 주차 폴더 안에 문제별 폴더를 생성합니다.
- 문제 폴더 안에는 본인 이름 파일명으로 소스코드를 업로드합니다.

예시:
- `week12/01_가장긴팰린드롬/soyun.cpp`
- `week12/02_거스름돈/soyun.cpp`

## 파일명 규칙

- 파일명은 **본인 이름 + 확장자** 형식으로 작성합니다.

예시:
- `soyun.cpp`

## Commit 규칙

커밋은 자유롭게 진행해도 됩니다.  
별도의 커밋 횟수 제한은 없으며, 작업 과정에 맞게 작성하면 됩니다.

## PR 규칙

### PR 제목 형식

- `이름 / 주차 / 문제 수`

예시:
- `soyun / week12 / 5문제`

## PR 제출 방법

- `main` 브랜치의 최신 내용을 pull 받습니다.
- 본인 이름 브랜치에서 문제를 풉니다.
- `add / commit / push` 를 진행합니다.
- GitHub에서 `main` 브랜치로 PR을 생성합니다.
- 팀장이 확인 후 merge 합니다.

## 주의 사항

- `main` 브랜치에 직접 push 하지 않습니다.
- 본인 브랜치에서만 작업합니다.
- 기존 폴더 및 다른 사람 파일은 수정하지 않습니다.
- 반드시 정해진 폴더 구조를 지켜 업로드합니다.
- 불필요한 파일은 업로드하지 않습니다.  
  예: `.vscode/`, 실행파일, 빌드 결과물 등

  ## 권장 `.gitignore`

```gitignore
.vscode/
*.exe
*.out
*.o
*.obj
```

## 벌칙 / 미제출 기준

- 매주 **수요일 오후 6시까지** PR 미제출 시 미제출 처리합니다.

## 스터디 목표

- 꾸준한 문제 풀이 습관 만들기
- 다양한 풀이 방식 공유하기
- PR과 코드 리뷰에 익숙해지기
- 코딩 테스트 대비 실전 감각 기르기
